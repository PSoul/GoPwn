## 平台 Bug

### 13. P0: execute_code 100% 失败 — LLM 传自然语言给 code 参数

**现象:** 所有项目中 execute_code 调用全部返回 `SyntaxError: Unexpected identifier 'Node'`

**根因:** LLM planner 在 action 字段写 "执行可运行 Node.js 脚本抓取首页..."，execution-worker 的 `buildToolInput()` 将 action 直接映射到 code 参数（line 199: `input[name] = action`），导致 Node.js 收到自然语言文本当代码执行。

**修复方向:** 
1. 在 planner prompt 中明确说明：当使用 execute_code 工具时，action 字段必须包含可执行的 JavaScript 代码，而非自然语言描述
2. 或在 buildToolInput 中增加 code 参数的校验/转换逻辑

**影响:** 所有需要动态脚本探测的场景完全瘫痪，包括 Redis/MongoDB 协议探测、自定义 HTTP 请求等

---

### 14. P1: 去重严重失效 — 225 个 findings 中约 40% 是重复

**现象:** DVWA 的 "Cookie 安全属性缺失" 出现 7+ 次，"Server 版本暴露" 出现 5+ 次。

**根因:** 
- 每个 round 的 analyzer 独立产出 findings，不知道之前已有相同发现
- `normalizeTitle()` 模糊匹配不够强：同一问题的中英文标题（"Cookie Security Attributes Missing" vs "Cookie 安全属性缺失"）不匹配
- 不同 target 路径（`/login.php` vs `/setup.php` vs `/`）的相同类型问题视为不同

**修复方向:**
1. analyzer prompt 中注入已有 findings 列表，让 LLM 主动避免重复
2. 增强去重：按 (问题类型 + 根目标域) 去重，而非 (精确标题 + 精确路径)

---

### 15. P1: tcp_banner_grab 无法探测 Redis/MongoDB 等需主动交互的服务

**现象:** Redis(6379) 和 MongoDB(27017) 的 banner grab 结果全是 "无 Banner 返回"

**根因:** tcp_banner_grab 只做被动连接+读取，但 Redis 需发送 `PING\r\n`，MongoDB 需发送 isMaster 命令才能获得响应

**修复方向:** 在 MCP server 中增加常见协议的主动探测逻辑（发送最小化探针）

---

### 16. P2: fscan.exe 和 httpx.exe 二进制文件缺失 (ENOENT)

**现象:** `spawn mcps/fscan-mcp-server/bin/fscan.exe ENOENT`，`spawn mcps/httpx-mcp-server/bin/httpx.exe ENOENT`

**影响:** 端口扫描和 HTTP 技术检测功能完全不可用

**修复方向:** 重新下载/编译二进制文件到正确路径

---

### 17. P2: http_batch 参数映射失败

**现象:** MCP error -32602: `requests` 参数为 undefined

**根因:** buildToolInput 不知道如何构造 `requests` 数组参数
