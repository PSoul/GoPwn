**截图总数：** 56 张（`screenshots/` 目录）
**审查方法：** Playwright 自动截图 + 代码静态分析
**审查日期：** 2026-03-31

## P0 — 阻塞可用性（0 项）

**未发现 P0 级问题。** 所有 27 个页面在 desktop 和 mobile 视口下均能正常渲染和使用，核心流程（登录→创建项目→查看结果→设置管理）完整可走通。

## P1 — 影响体验（8 项）

### P1-1. Loading 状态覆盖不全

**问题：** 全站仅有 2 个 `loading.tsx`（控制台顶层 + 项目详情），其余 14 个子路由（settings/*、vuln-center、assets、approvals 等）均回退到控制台顶层的"4 列指标卡 + 5 行表格"骨架屏，与实际页面布局完全不匹配。

**影响：** 用户切换到设置页时看到仪表盘式骨架屏，造成视觉闪烁和认知混乱。

**建议：**
- 为 `settings/` 路由组添加 `loading.tsx`（左侧导航 + 右侧内容区骨架）
- 为 `vuln-center`、`assets`、`approvals` 添加匹配各自布局的 `loading.tsx`
- 优先级：settings > approvals > vuln-center > assets

### P1-2. 页面切换无过渡动画（闪白）

**问题：** 路由切换时整个内容区直接替换，无任何过渡效果，产生明显的白屏闪烁（尤其在深色模式下更刺眼）。

**影响：** 用户感知上"卡了一下"，降低流畅感。

**建议：**
- 使用 CSS fade-in 动画 + `key={pathname}` 触发路由切换重挂载
- 最简方案：在 `app-shell.tsx` 的 `{children}` 外包一层 fade-in 动画

### P1-3. Operations 页面信息密度过高（移动端尤其严重）

**问题：** `operations` 页面在项目有调度任务时，单页展示所有任务卡片 + 详细日志 + 状态徽章，fullPage 截图高达 443KB（mobile），内容纵向延伸极长。

**影响：** 移动端用户需要大量滚动，难以快速定位关键信息。

**截图证据：** `projects_proj-*_operations-mobile.png` (443KB, 对比其他页面平均 50-80KB)

**建议：**
- 底部面板默认折叠，仅显示标题，点击展开详情
- 添加"跳转到顶部"浮动按钮

### P1-4. MCP 工具设置页内容过长

**问题：** `settings/mcp-tools` 页面展示所有 MCP 工具的完整配置（包括参数 schema），desktop 截图 551KB、mobile 520KB，是全站最大的页面。

**影响：** 大量工具配置信息一字排开，查找特定工具困难。

**截图证据：** `settings_mcp-tools-desktop.png` (551KB), `settings_mcp-tools-mobile.png` (520KB)

**建议：**
- 次要区块用 Accordion 折叠
- 保留统计概览和工具列表始终可见

### P1-5. 审批策略页信息架构复杂

**问题：** `settings/approval-policy` 页面同时展示策略规则列表 + 规则编辑表单 + 条件配置，desktop 截图已达 175KB，移动端 156KB。

**截图证据：** `settings_approval-policy-desktop.png`, `settings_approval-policy-mobile.png`

**建议：**
- 规则列表和编辑区分离为两步操作（列表页 → 编辑页/抽屉）
- 或使用 Sheet/Drawer 组件做侧边编辑面板

### P1-6. 破坏性操作缺少确认对话框

**问题：** `AlertDialog` 仅在项目归档操作中使用。以下操作缺少二次确认：
- 调度任务的取消/停止
- 审批决策（通过/拒绝）

**影响：** 误触可导致正在运行的任务被终止或审批策略被意外修改。

**建议：**
- 所有"不可逆"或"影响运行中任务"的操作添加 AlertDialog 确认
- 已有 `alert-dialog.tsx` 组件，复用即可

### P1-7. 表单仅顶部 Alert 提示错误，缺少字段级定位

**问题：** 虽然 `components/ui/form.tsx` 支持 `FormMessage` 字段级错误，但实际表单页面主要使用顶部统一 Alert 展示错误。

**建议：**
- 将表单迁移到 react-hook-form + FormField 模式
- 错误时自动 scroll-to 并 focus 第一个出错字段

### P1-8. 登录页验证码交互不直观

**问题：** 登录页验证码展示在一个按钮上，用户需要看按钮文字再手动输入到验证码输入框。

**建议：**
- 验证码改为图片展示（更直观的视觉区分）
- 或直接使用点击式验证替代文本输入

## P2 — 打磨细节（9 项）

### P2-1. 圆角值不统一

**问题：** 全站混用 16+ 种圆角：`rounded-[36px]`、`rounded-[32px]`、`rounded-[28px]`、`rounded-[24px]`、`rounded-[22px]`、`rounded-[20px]`、`rounded-[18px]` 等，没有统一的设计 token。

**建议：** 定义 4 个语义化圆角 token（hero/card/panel/item），全站统一引用。

### P2-2. 深色模式仅截了 4 个页面

**建议：** 补充全站深色模式截图，重点关注设置页面、登录页、审批策略页。

### P2-3. Dashboard 空数据时指标卡缺少引导

**问题：** Dashboard 4 个指标卡在无数据时显示"0"，缺少引导性 CTA。

**建议：** 当所有指标为 0 时，显示一个引导性的 onboarding 卡片。

### P2-4. 移动端侧边栏深层导航返回路径

**建议：** 面包屑添加 `overflow-x-auto` 横向滚动，或在移动端用"← 返回"按钮替代完整面包屑。

### P2-5. Settings LLM 页面过于空旷

**截图证据：** `settings_llm-desktop.png` (42KB — 全站最小的页面)

**建议：** 增加使用统计或连接测试按钮，填充内容区。

### P2-6. 项目 AI-Logs 页面空状态文案可优化

**建议：** 空状态添加说明文案 + 开始执行的 CTA 按钮。

### P2-7. 项目结果三子页面空状态布局高度一致

**问题：** domains/network/findings 三个结果页空状态几乎完全一样，无法从视觉上区分。

**建议：** 各结果页添加不同的图标和差异化文案。

### P2-8. Flow 流程图页面需手动交互测试

**建议：** 需要手动或 Playwright 交互脚本测试拖拽、缩放等操作。

### P2-9. 项目编辑页与新建页表单不完全对称

**建议：** 确认编辑页与新建页共用同一个表单组件。
