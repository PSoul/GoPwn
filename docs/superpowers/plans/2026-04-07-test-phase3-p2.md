## Task 1: 创建 mock-event-source.ts 测试助手

**Files:**
- Create: `tests/helpers/mock-event-source.ts`

- [ ] **创建 EventSource mock 类**

为 hooks 测试提供可控的 EventSource 模拟，支持手动触发 open/message/error 事件。

```typescript
// mock EventSource：可控触发 onopen / onmessage / onerror
// 提供 instances 数组追踪所有实例（便于测试中获取最近创建的 ES）

export class MockEventSource {
  static instances: MockEventSource[] = []
  url: string
  onopen: (() => void) | null = null
  onmessage: ((e: { data: string }) => void) | null = null
  onerror: ((e: unknown) => void) | null = null
  readyState = 0 // CONNECTING
  closed = false

  constructor(url: string) {
    this.url = url
    MockEventSource.instances.push(this)
  }

  close() {
    this.readyState = 2 // CLOSED
    this.closed = true
  }

  // ---- 测试触发器 ----
  triggerOpen() {
    this.readyState = 1 // OPEN
    this.onopen?.()
  }
  triggerMessage(data: string) {
    this.onmessage?.({ data })
  }
  triggerError(err?: unknown) {
    this.onerror?.(err ?? new Error("connection lost"))
  }

  static reset() {
    MockEventSource.instances = []
  }
}
```

## Task 4: React Hook — use-project-events.test.ts

**Files:**
- Create: `tests/unit/hooks/use-project-events.test.ts`
- Test: `lib/hooks/use-project-events.ts`

- [ ] **Step 1: 测试文件框架**

```typescript
import { describe, it, expect, vi, beforeEach, afterEach } from "vitest"
import { renderHook, act } from "@testing-library/react"
import { MockEventSource } from "../../helpers/mock-event-source"
import { useProjectEvents } from "@/lib/hooks/use-project-events"

// 全局替换 EventSource
beforeEach(() => {
  MockEventSource.reset()
  vi.stubGlobal("EventSource", MockEventSource)
})
afterEach(() => {
  vi.unstubAllGlobals()
})
```

- [ ] **Step 2: 实现 7 个测试用例**

| # | 测试 | 关键断言 |
|---|------|----------|
| 1 | 连接建立 | `renderHook(() => useProjectEvents("p1"))` → MockEventSource 实例化，url 包含 `/api/projects/p1/events`；triggerOpen → `connected === true` |
| 2 | 接收事件 | triggerMessage(`JSON.stringify({type:"lifecycle_changed", projectId:"p1", timestamp:"...", data:{}})`) → `lastEvent.type === "lifecycle_changed"` |
| 3 | onEvent 回调 | 传入 `onEvent` spy → triggerMessage → spy 被调用，参数为解析后的 event |
| 4 | JSON 解析错误容错 | triggerMessage("not-json") → 不抛异常，lastEvent 不变 |
| 5 | 断线 3s 重连 | triggerError → `connected === false`；vi.advanceTimersByTime(3000) → 新的 MockEventSource 实例被创建 |
| 6 | projectId 变更 | 初始 "p1"，rerender "p2" → 旧 ES close，新 ES url 包含 "p2" |
| 7 | 卸载清理 | unmount → ES.close 被调用 |

注意：测试 5 需要 `vi.useFakeTimers()` / `vi.useRealTimers()`。测试 6 使用 `renderHook` 返回的 `rerender`。

    const res = await request.get(`/api/projects/${projectId}/report-export`, {
      headers: { Cookie: `pentest_token=${tokenCookie.value}` },
    })
    // 新项目无数据时应返回 200（空报告）或 404
    expect([200, 404]).toContain(res.status())
