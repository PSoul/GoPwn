# 项目详情页重构设计

## 背景

当前项目详情页存在三个问题：
1. 顶部导航的 4 个 tab（域名/站点/端口/漏洞）路由页面在 Phase 24.5 被删除，点击 404
2. 工具执行失败的日志被 analyzer 当作漏洞存储（已修复：execution-worker.ts 不再将失败输出送 analyzer）
3. 资产列表所有类型混在一张表里，缺乏分类

## 设计目标

- 重建项目详情页信息架构，消除 404 和功能重叠
- 资产按发现链层级分类展示（DNS → 网络 → 应用）
- 漏洞独立成页，配合详情页展示完整证据链
- IP 资产提供钻取详情页，聚合端口/服务/漏洞/Web应用

## 信息架构

```
顶部导航：概览 | 资产 | 漏洞 | 执行控制 | AI日志

概览页（仪表盘）
├── 项目状态栏（轮次、生命周期、阶段）
├── 安全发现摘要（按严重级别计数卡片，点击 → 漏洞页）
├── 资产发现摘要（按类型计数卡片，点击 → 资产页对应 tab）
└── 最近活动（最近几条工具执行结果/发现）

资产页（3 个子 tab）
├── 域名
├── 主机与端口
└── Web 与 API
    └→ IP 详情页（从表格中 IP 链接点击进入）

漏洞页
├── 漏洞列表
└→ 漏洞详情页（点击行进入）

执行控制（已有，保留）
AI 日志（已有，保留）
```

## 顶部导航

从当前 7 个 tab 简化为 5 个：

| Tab | 路由 | 说明 |
|-----|------|------|
| 概览 | `/projects/[id]` | 摘要仪表盘 |
| 资产 | `/projects/[id]/assets` | 全部资产，内部 3 个子 tab |
| 漏洞 | `/projects/[id]/findings` | 全部安全发现 |
| 执行控制 | `/projects/[id]/operations` | 已有 |
| AI 日志 | `/projects/[id]/ai-logs` | 已有 |

删除：域名、站点、端口（合并入资产页子 tab）。
删除：概览页内嵌的 tab 系统（漏洞/资产/日志），改为顶部统一导航。

## 概览页

概览页是仪表盘，不展示完整列表，只做入口。

### 项目状态栏
已有组件，保留：当前轮次、生命周期状态、当前阶段。

### 安全发现摘要
按严重级别的计数卡片，例如 "2 High / 5 Medium / 11 Low / 47 Info"。
点击任意级别 → 跳转漏洞页（可带筛选参数）。

### 资产发现摘要
按类型的计数卡片，例如 "12 域名 / 8 主机 / 45 端口 / 23 Web应用"。
点击 → 跳转资产页对应子 tab。

### 最近活动
最近 5-10 条工具执行结果或新发现，快速了解进展。

## 资产页

资产页包含 3 个子 tab，按网络层级划分。每个子 tab 的计数显示在 tab 标签上。

子 tab 通过 URL query 参数切换（不创建独立路由）：
- `/projects/[id]/assets` — 默认显示域名 tab
- `/projects/[id]/assets?tab=domains` — 域名
- `/projects/[id]/assets?tab=hosts` — 主机与端口
- `/projects/[id]/assets?tab=web` — Web 与 API

### 子 Tab 1：域名

展示 DNS 层面发现的所有域名资产。

| 列名 | 说明 |
|------|------|
| 域名 | 域名全称 |
| 类型 | 根域名 / 子域名 |
| 解析 IP | 解析到的 IP 地址，可点击跳转 IP 详情页；未解析则灰色标注 |
| 发现时间 | 首次发现时间 |

数据量大时正常分页。

### 子 Tab 2：主机与端口

展示网络层面发现的所有 IP + 开放端口 + 服务识别。一行一个端口。

| 列名 | 说明 |
|------|------|
| 主机 | IP 地址，可点击跳转 IP 详情页 |
| 端口 | 端口号 |
| 协议 | TCP / UDP |
| 服务 | 从 Banner 探测自动识别的服务类型（HTTP/SSH/MySQL/Redis 等） |
| Banner/版本 | 服务 Banner 或版本信息 |
| 发现时间 | 首次发现时间 |

MySQL、Redis、SSH、MongoDB 等非 HTTP 服务都在这里展示。
可按主机排序，同一主机的端口连续展示。

### 子 Tab 3：Web 与 API

展示应用层面发现的 HTTP 可达资源。

| 列名 | 说明 |
|------|------|
| URL | 完整 URL |
| 类型 | Web 应用 / API 端点 |
| 标题/描述 | 页面标题或端点描述 |
| 技术栈 | 识别到的技术（WordPress/nginx/React 等） |
| 状态码 | HTTP 响应状态码 |
| 发现时间 | 首次发现时间 |

只有 HTTP/HTTPS 可达的资源出现在这里。

## IP 详情页

路由：`/projects/[id]/assets/ip/[assetId]`

从资产页表格中点击 IP 地址链接进入。

### 入口
- 主机与端口 tab → 点击"主机"列的 IP
- 域名 tab → 点击"解析 IP"列的 IP

## 漏洞页

### 漏洞列表

| 列名 | 说明 |
|------|------|
| 漏洞标题 | 漏洞名称 |
| 严重级别 | critical / high / medium / low / info，带颜色徽章 |
| 影响目标 | 受影响的资产地址 |
| 发现时间 | 首次发现时间 |

无"状态"列 — 这是自动化平台，LLM 确认后才写入，无需人工确认流程。
点击行跳转漏洞详情页。

### 漏洞详情页

路由：`/projects/[id]/vuln/[findingId]`（已有路由，需重构内容）

```
┌─────────────────────────────────────┐
│ ← 返回漏洞列表                       │
│                                     │
│ Redis 未授权访问            [High]   │
│ 影响目标: 10.0.1.1:6379             │
│ 发现时间: 2026/4/5 14:32            │
├─────────────────────────────────────┤
│ 漏洞描述                            │
│ （summary 字段）                     │
├─────────────────────────────────────┤
│ 修复建议                            │
│ （recommendation 字段）              │
├─────────────────────────────────────┤
│ 验证证据 (PoC)                      │
│ 代码块（可复制）                     │
│ 执行输出块                          │
├─────────────────────────────────────┤
│ 原始证据（默认折叠）                  │
│ 工具名 + 完整 rawOutput             │
└─────────────────────────────────────┘
```

区块说明：
1. **头部** — 标题、严重级别徽章、影响目标、发现时间
2. **漏洞描述** — Finding.summary，LLM 生成的漏洞说明
3. **修复建议** — Finding.recommendation
4. **验证证据** — 关联的 Poc 记录：代码 + 语言 + 执行输出
5. **原始证据** — 关联 Evidence 的完整工具输出，默认折叠

## 页面跳转关系

```
域名 tab [解析IP列] ─────→ IP 详情页
主机与端口 tab [主机列] ──→ IP 详情页
IP 详情页 [关联漏洞] ────→ 漏洞详情页
IP 详情页 [关联Web应用] ─→ 资产页 Web与API tab
漏洞列表 [行点击] ───────→ 漏洞详情页
漏洞详情页 [返回] ───────→ 漏洞列表
IP 详情页 [返回] ────────→ 主机与端口 tab
概览页 [安全发现摘要] ───→ 漏洞页
概览页 [资产发现摘要] ───→ 资产页对应子 tab
```
