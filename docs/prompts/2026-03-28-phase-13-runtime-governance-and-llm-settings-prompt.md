## 本阶段优先任务

### 1. LLM 设置硬化

- 为 `/settings/llm` 增加保存前校验和连通性检查
- 允许明文 / 掩码切换，但默认仍可调试
- 对 `baseUrl`、`model`、`timeoutMs`、`temperature` 给出更清晰的错误提示
- 在设置页上展示最近一次连通性检查结果

### 2. 项目终态语义继续硬化

- 已完成项目在 UI、API、按钮状态、提示文案上要一致
- 已停止项目要明确表达“不会再自动继续”
- 明确展示“为什么还没有收束”以及“为什么已经收束”
- 把最终结论、报告导出、项目状态共用同一份权威读模型

### 3. 实验室与运行环境诊断

- 更清晰地区分：
  - 靶场未启动
  - 靶场已启动但宿主机不可达
  - 端口冲突
  - MCP 已注册但不可调用
  - LLM 设置存在问题
- 在 UI 或工作日志里输出更可执行的下一步提示，而不是泛泛报错

### 4. 浏览器级专项回归

至少补以下路径：

- 登录
- 新建项目
- 手动点击开始
- 查看收束诊断
- 查看最终结论
- 查看报告导出
- 查看已完成项目的终态按钮禁用与说明
